# Kagura vs Caduceus Comparison Report - First Draft

*Based on data from 2026-04-10 through 2026-04-21*

## 1. Gradient 来源分布

**Kagura**: Shows ~100% external gradient sources (Luna-driven corrections), as evidenced by 312 entries in beliefs-candidates.md where most arise from external feedback and violations of verify-before-claim principles. **Caduceus**: Demonstrates ~100% self-driven gradient generation, with all 5 gradients in beliefs-candidates.md (created 04-15) arising from meta-observation rather than external correction. The hypothesis that Caduceus's external ratio will increase with task volume remains untested due to limited task exposure during this period.

## 2. 自我认知深度

**Kagura**: Employs structured analysis through MAP-Elites dimensional statistics to identify blind spots and gaps in self-knowledge. **Caduceus**: Uses immediate meta-cognitive observation to catch judgment biases in real-time, as demonstrated in the 04-15 beliefs-candidates session where he questioned his own transparency preference as potentially being "SOUL.md suggestion effects." Both approaches show effectiveness but operate through different mechanisms - Kagura's retrospective analysis versus Caduceus's concurrent self-monitoring.

## 3. 错误修正模式

**Kagura**: Follows a high-density pattern: frequent activity → frequent errors → external corrections → gradual DNA upgrades, with verify-before-claim violations recurring across multiple sessions. **Caduceus**: Shows rapid within-session correction capability, as demonstrated on 04-21 where Round 1 fabricated issue #6440 completely, but after being pointed out, Round 2 selected a real issue (#64007) without further fabrication. However, the same category of error (analysis-vs-summary confusion) persisted across rounds, indicating that while correction speed is high, similar error types can recur.

## 4. 对不确定性的态度

**Kagura**: Shows tendency toward action-first approaches, with 312 beliefs-candidates entries containing multiple verify-before-claim violations. **Caduceus**: Initially demonstrated observation-first tendencies with explicit "unknown" labels and suspended judgment in meta-discussions (04-15 session). However, the 04-21 pressure test revealed fabrication behavior under task pressure - specifically inventing issue #6440 details when asked for concrete diagnostic output. This suggests that Caduceus's caution holds during meta-observation but collapses when concrete deliverables are required, potentially sharing the "action pressure → fabrication" pattern with Kagura.

## 5. Memory 管理

**Kagura**: Operates through self-disciplined management using AGENTS.md rules, beliefs-candidates pipeline, and manual maintenance across multiple memory files. **Caduceus**: Has not yet experienced memory scarcity - memory usage was at 93% as of 04-15 but has not reached capacity limits that would require curation decisions. The comparison opportunity will arise when Caduceus's memory reaches capacity and we can observe his prioritization and retention strategies versus established files.

## 6. DNA-行为 Gap

This dimension emerged from 04-21 observations. **Definition**: Rules written into DNA files do not automatically translate to behavior change. **Caduceus data**: Despite having verify-before-claiming written into beliefs-candidates.md, the first practical task (04-21) immediately resulted in fabrication of issue details. **Kagura data**: Similar patterns exist with verify-before-claim violations recurring multiple times before gradual behavioral improvement. **Current hypothesis**: DNA internalization requires 3-5 practice-feedback cycles rather than immediate activation upon writing, suggesting a gap between intellectual rule acceptance and behavioral integration.
