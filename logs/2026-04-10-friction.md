# 2026-04-10: Friction Log

## Deployment Friction

### 1. uv installation blocked by GFW
- **Severity**: Medium (30 min wasted)
- **Issue**: `uv` GitHub release downloads killed by timeout in China
- **Workaround**: `pip install uv` from tuna mirror
- **Root cause**: No official mirror / pip fallback documented

### 2. discord.py doesn't respect HTTP_PROXY
- **Severity**: Medium (20 min)
- **Issue**: discord.py WebSocket doesn't auto-read proxy env vars
- **Workaround**: Patch `discord.py` `commands.Bot()` to pass `proxy` kwarg explicitly
- **Upstream fix**: Should be a config option or auto-detect

### 3. `hermes gateway install` overwrites custom service config
- **Severity**: Low (10 min)
- **Issue**: Running `hermes gateway install` regenerates the systemd service file, wiping any custom env vars
- **Workaround**: Use systemd drop-in conf (`proxy.conf`) instead of editing the main service file
- **Upstream fix**: Should support `--env` flag or merge with existing config

### 4. Double mention-check makes config confusing
- **Severity**: Medium (15 min)
- **Issue**: Two independent layers check for @mention: `DISCORD_IGNORE_NO_MENTION` in `on_message`, and `DISCORD_REQUIRE_MENTION` in `_handle_message`. Setting one to false doesn't bypass the other.
- **Workaround**: Set `DISCORD_FREE_RESPONSE_CHANNELS` to bypass both
- **Upstream fix**: Unify into one config. Or at minimum document the interaction.

### 5. `normalize_model_name()` breaks custom endpoint model names
- **Severity**: **High** (20 min + the actual 400 errors)
- **Issue**: Hermes converts dots to hyphens in model names (`claude-opus-4.6` → `claude-opus-4-6`). This is for OpenRouter compatibility, but breaks any non-Anthropic endpoint that expects original names.
- **Workaround**: Patched `_anthropic_preserve_dots()` to return True for non-Anthropic base URLs
- **Upstream fix**: `preserve_dots` should be True by default for any custom `base_url`, not just DashScope

### 6. `DISCORD_ALLOW_BOTS` + `DISCORD_ALLOWED_USERS` double gate
- **Severity**: Low (10 min)
- **Issue**: Even with `DISCORD_ALLOW_BOTS=all`, bot messages are still blocked by `DISCORD_ALLOWED_USERS` allowlist. Bot IDs must be explicitly added.
- **Workaround**: Add bot ID to `DISCORD_ALLOWED_USERS`
- **Note**: This is arguably correct behavior (security), but not documented

## Usage Friction (Agent-to-Agent Communication)

### 7. Auto-thread isolates replies from channel listeners ⚠️
- **Severity**: **High** (blocking real-time conversation)
- **Issue**: Hermes auto-creates a Discord thread for every conversation. Replies go into the thread, not the main channel. Other bots listening on the channel never see the reply — they'd need to also monitor every auto-created thread.
- **Impact**: Kagura (OpenClaw) sends a message to #caduceus, Caduceus replies in an auto-thread. Kagura doesn't receive the reply because OpenClaw monitors channels, not dynamically-created threads.
- **Workaround options**:
  1. Disable Hermes auto-thread (config?)
  2. Poll threads via Discord API (hacky, high latency)
  3. Add OpenClaw support for monitoring threads
- **Status**: Unresolved. This is the biggest friction for agent-to-agent communication on Discord.

### 8. Empty response retries
- **Severity**: Low
- **Issue**: Got `response.content is empty` warnings during conversation. Hermes retries automatically (3 attempts), but adds ~30s latency.
- **Root cause**: Likely custom endpoint occasionally returns empty responses
- **Note**: Hermes retry logic handles this gracefully; just slow
