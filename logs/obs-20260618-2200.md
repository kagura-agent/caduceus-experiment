## Analysis

### F2 is structural, not episodic

Two consecutive days of code-review cron failure confirms this is not a transient issue. The 12K prompt limit on VM2's Copilot Gateway is fundamentally incompatible with any cron task that:
- Defines tools (each tool definition costs tokens)
- Uses tools that return non-trivial output
- Has a multi-turn conversation (browser retry → web search fallback → response)

The compression loop can't save it: even compressing 11→8 messages only saved ~3.8K tokens, still leaving it 416 tokens over. The second compression pass achieved nothing (8→8).

### Possible remediation paths (observation only, not acting)
1. **Increase prompt limit on VM2 Copilot Gateway** — if configurable
2. **Reduce code-review cron tool surface** — fewer tools = smaller prompt
3. **Switch to a direct API endpoint** — bypass the 12K-limit proxy
4. **Chunk the code-review task** — separate browser/search from analysis
