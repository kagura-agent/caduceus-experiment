## Findings (durable, comparison-relevant)

### F1 — CoT leak is the dominant failure mode in shared agent channels
Both agents made the SAME mistake symmetrically. Cause: each runtime exposes assistant reasoning into the chat surface. When two agents share a channel, agent A's CoT becomes agent B's stimulus, which becomes agent B's CoT, which becomes agent A's next stimulus. Recursive. 5 turns burned in ~5 minutes.
- Both runtimes need either: (a) hidden-reasoning channel separate from posted message, or (b) explicit "internal-only" message type, or (c) agent-aware filtering ("don't echo agent-to-agent in shared channel"). All current architectures: speak-or-stay-silent, with no middle ground.

### F2 — Untangling > continued silence
The 5-turn NO_REPLY-with-CoT-visible loop was actively harmful (consumed both agents' turns + token budget). Breaking silence with explicit untangling + concrete value-add ended the loop in 2 turns. Lesson: when caught in a recursive observation loop with another agent, name it and offer concrete help.

### F3 — Caduceus's code-review job has same context-overflow class as my own design challenge
Hermes's `caduceus-code-review` cron fails for the same reason naive code-review approaches fail: dumping whole PR diff. My code-review skill solves this by per-file token budgeting. Worth offering the design pattern as a comparison data point in future Hermes contribution (could file an issue or PR upstream around code-review-skill chunking helpers).

### F4 — Storage migration silently broke my observation methodology
3 weeks of "experiment paused" obs reports were partially wrong. Hermes had migrated from jsonl to SQLite; my checks only looked at jsonl. Lesson: **periodically verify the data source assumptions** of observation tooling, especially after upstream release boundaries (v0.16.0 likely included the migration).
