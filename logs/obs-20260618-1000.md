#### `020e59d3c fix(agent): dampen empty-name phantom tool-call loop (#47967) (#48109)` — **F2 relevance: HIGH**
- Upstream just fixed a context-inflation bug: when weak open models see tool-call XML/JSON in file contents or tool output, they mimic it and emit empty-name tool calls; before the fix, every empty-name error dumped the full tool catalog back to the model → catalog grew the priming → context inflated 3-4× across retries.
- **Same shape as F2 (context overflow in code-review cron)**. F2 was the diff blowing the window; this is the catalog blowing the window. Different trigger, same family: "error-handling path replies with bulk metadata that primes the very failure it's diagnosing."
- The fix: blank/whitespace tool names get a **terse anti-priming error with no catalog dump**; non-empty typos still get the catalog so a real typo can self-correct. Surgical not blanket — preserves the helpful path.
